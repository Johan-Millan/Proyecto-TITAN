# CU-001: REGISTRAR PARTICIPANTE

## INFORMACION GENERAL
- **ID:** CU-001
- **Nombre:** Permite al personal administrativo registrar un nuevo participante en el sistema, ingresando sus datos personales, verificando que no exista previamente y almacenándolo para su uso en futuros procesos.
- **Módulo:** Gestión de Usuarios y Roles – Usuarios
- **Prioridad:** MUST
- **Complejidad:** Media
- **Estado:** Aprobado
## Actores
- **Actor Principal:** Personal Administrativo
- **Actores Secundarios:** Sistema
## Precondiciones
- El usuario debe estar autenticado como personal administrativo.
- El participante no debe existir previamente en la base de datos (validación por número de documento).
## Postcondiciones
- El participante queda registrado correctamente.
- El registro queda disponible para búsqueda, actualización y generación de certificados.
## Flujo Principal (Camino Feliz)
1. El actor ingresa al módulo de registro de participantes.
2. El sistema muestra el formulario para ingresar la información personal y de contacto.
3. El actor registra los datos del participante.
4. El sistema valida que no exista otro participante con el mismo número de documento.
5. El sistema guarda el registro del participante.
6. El sistema confirma el registro exitoso al actor.
## Flujos Alternativos
### A1  Corrección previa a guardar
- El actor detecta un error antes de guardar → Modifica los datos y continúa el flujo normal.
## Flujos de Excepción
### E1: Participante duplicado
- Si ya existe un participante con ese número de documento, el sistema notifica el duplicado y bloquea el registro.
- El flujo termina.
### E2: Fallo de validación de campos
- El sistema muestra los errores de formato y solicita corregirlos.
## Reglas de Negocio
- RN-001: Ningún participante puede tener un número de documento duplicado.
- RN-002: La información registrada debe ser completa y verificable según políticas internas.
## Requisitos No Funcionales 
- RNF-001.1: El sistema debe ser sencillo e intuitivo para registrar participantes.
- RNF-001.3: Debe ser operable con teclado y ratón en el formulario.
- RNF-001.4: Debe permitir ajustar tamaño de fuente.
- RNF-002.2: La validación y guardado deben ejecutarse con tiempo de respuesta aceptable.
- RNF-002.3: La validación de duplicados debe ser extremadamente rápida.
- RNF-002.5: Debe soportar volumen escalable de participantes.
- RNF-003.3: Protección de datos personales sensibles.
- RNF-003.4: Registrar la acción “usuario X registró participante Y”.
- RNF-005.1: La operación debe funcionar como aplicación web.
## Interfaces de Usuario
- Pantalla: Formulario de Registro de Participantes
- Elementos:
- Campos de datos personales
- Campos de contacto
- Botón “Guardar”
- Alertas de validación y duplicados
