Información General

ID: CU-006

Nombre: Registrar Incidente

Descripción breve: Permite registrar incidentes ocurridos en el centro de entrenamiento incluyendo detalles, responsables y evidencia.

Módulo: Reportes y Cumplimiento

Prioridad: MUST

Complejidad: Media

Estado: En Revisión

Actores

Actor Principal: Personal Administrativo

Actores Secundarios: Sistema (validación, auditoría)

Precondiciones

El usuario debe estar autenticado y autorizado para registrar incidentes. (REF-03)

El módulo de incidentes debe estar disponible. (REF-01)

Postcondiciones

El incidente queda registrado con un ID único. (REF-01)

La acción queda registrada en el sistema de auditoría. (REF-03)

Flujo Principal (Camino Feliz)

El actor accede al módulo “Registro de Incidentes”. (REF-01)

El sistema muestra un formulario para ingresar datos del incidente. (REF-02)

El actor ingresa la información requerida: fecha, responsable, descripción y evidencia. (REF-01)

El sistema valida los datos ingresados. (REF-04)

El sistema registra el incidente y genera un ID único. (REF-01)

El sistema almacena la evidencia asociada. (REF-01)

El sistema registra la acción en el módulo de auditoría. (REF-03)

El sistema confirma el registro exitoso. (REF-01)

Flujos Alternativos
A1 – Evidencia no cargada

3a. El actor decide registrar el incidente sin evidencia.

3b. El sistema permite continuar indicando que la evidencia puede agregarse después.
(REF-01)

A2 – Información incompleta pero no crítica

4a. El sistema detecta campos opcionales vacíos.

4b. El sistema advierte pero permite continuar.
(REF-04)

Flujos de Excepción
E1 – Error en los datos ingresados

4c. El sistema detecta campos obligatorios mal diligenciados.

4d. Notifica el error y solicita corrección.

4e. El actor regresa al paso 3.
(REF-04)

E2 – Falla al guardar el incidente

5a. El sistema presenta un error al intentar guardar.

5b. Notifica el problema y solicita reintentar.
(REF-05)

Reglas de Negocio
