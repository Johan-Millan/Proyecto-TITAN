# CU-011 – Corrección de Errores en Datos

## Información General
- **ID:** CU-011
- **Nombre:** Corrección de Errores en Datos
- **Descripción breve:** Permite al personal administrativo corregir directamente errores detectados en registros de participantes, instructores o incidentes.
- **Módulo:** Gestión de Usuarios y Roles
- **Prioridad:** MUST
- **Complejidad:** Media
- **Estado:** En Revisión
## Actores
- **Actor Principal:** Personal administrativo
- **Actores Secundarios:** Sistema
## Precondiciones
- El usuario debe estar autenticado y tener permisos de edición. 
- El registro a corregir debe existir en el sistema.
## Postcondiciones
- Los datos erróneos quedan corregidos y actualizados.
- La acción queda registrada en el historial de auditoría.
## Flujo Principal (Camino Feliz)
1. El usuario busca el registro que requiere corrección.
2. El sistema muestra los datos actuales del registro.
3. El usuario modifica los campos incorrectos.
4. El sistema valida los cambios y guarda la información actualizada.
5. La acción queda registrada en el historial de auditoría.
## Flujos Alternativos
### A1 – Corrección parcial:
- El usuario puede modificar solo algunos campos del registro sin afectar otros datos.
### A2 – Validación de formato:
- El sistema verifica que los datos modificados cumplan con los formatos requeridos antes de guardar.
## Flujos de Excepción
### E1 – Usuario no autorizado:
- El sistema bloquea la edición y muestra un mensaje de acceso denegado. 
### E2 – Error de validación de datos:
- El sistema muestra los errores y solicita corrección antes de guardar. 
## Reglas de Negocio
- RN-20: Solo el personal administrativo puede realizar correcciones en registros existentes.
- RN-21: Todas las modificaciones deben quedar registradas en el historial de auditoría.
## Requisitos No Funcionales
- RNF-001.1: Interfaz clara e intuitiva para la edición de registros.
- RNF-002.2: Validación y guardado de cambios con tiempo de respuesta aceptable.
- RNF-003.3: Protección de datos personales sensibles durante la modificación.
- RNF-003.4: Registro automático de la acción en auditoría.
## Interfaces de Usuario
- Pantalla de búsqueda y selección de registro.
- Formulario de edición de datos.
- Alertas de validación y confirmación de cambios.
## Referencias
- RF-002.5: Permite corregir directamente errores en la entrada de datos.
- HU-009: Corrección de errores en entrada de datos.
- RNF-001.1, RNF-001.3: Usabilidad.
- RNF-003.4: Auditoría.
