CU-005: REGISTRAR INCIDENTE
INFORMACIÓN GENERAL

ID: CU-005

Nombre: Permite registrar formalmente un incidente ocurrido durante las actividades del centro.

Módulo: Gestión de Incidentes

Prioridad: MUST

Complejidad: Media

Estado: Aprobado

Actores

Actor Principal: Personal Administrativo

Actores Secundarios: Sistema

Precondiciones

El actor debe estar autenticado.

El incidente debe corresponder a un evento real ocurrido en el centro.

Debe existir registro previo del participante o instructor involucrado (si aplica).

Postcondiciones

El incidente queda registrado con su ID único.

La información queda disponible para consulta y auditoría.

Se genera un registro en el historial de auditoría.

Flujo Principal (Camino Feliz)

El actor ingresa al módulo de registro de incidentes.

El sistema muestra el formulario de registro.

El actor ingresa la información del incidente (fecha, lugar, descripción, involucrados).

El actor adjunta evidencia si corresponde (imágenes, documentos).

El sistema valida que los campos obligatorios estén completos.

El sistema guarda el incidente en la base de datos.

El sistema genera un ID único para el incidente.

El sistema confirma el registro exitoso.

Flujos Alternativos

A1 – Sin evidencia adjunta

En el paso 4, el actor decide no adjuntar evidencia.

El sistema continúa desde el paso 5 sin requerir archivo.

Flujos de Excepción

E1 – Datos incompletos

5a. El sistema detecta campos obligatorios vacíos.

5b. Se muestra un mensaje solicitando completar la información.

El actor corrige y regresa al paso 3.

E2 – Formato de archivo no permitido

4a. El sistema detecta que el archivo adjunto tiene un formato inválido.

4b. Notifica al actor y bloquea la carga.

Reglas de Negocio

RN-006: Todo incidente debe tener una fecha, descripción y responsable.

RN-007: Las evidencias deben ser almacenadas cumpliendo normas de protección de datos.

Requisitos No Funcionales

RNF-001.1: La interfaz debe ser sencilla para registrar incidentes de forma rápida.

RNF-001.3: El formulario debe ser completamente navegable con teclado.

RNF-002.2: El guardado debe realizarse con tiempos de respuesta aceptables.

RNF-002.5: El sistema debe soportar almacenamiento creciente de incidentes y evidencias.

RNF-003.3: Protección de datos personales e información sensible.

RNF-003.4: Todo registro debe reflejarse en auditoría.

RNF-005.1: Funcionamiento como aplicación web.
