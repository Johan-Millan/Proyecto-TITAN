CU-003 – Actualizar Información de Participante
Información General

ID: CU-003

Nombre: Actualizar Información de Participante

Módulo: Gestión de Usuarios y Roles

Prioridad: MUST

Complejidad: Media

Estado: En Revisión

Actores

Actor Principal: Personal Administrativo

Actores Secundarios: Sistema (auditoría y validaciones automáticas)

Precondiciones

El participante debe existir en el sistema. (REF-01)

El usuario debe haber iniciado sesión con permisos adecuados. (REF-03)

Postcondiciones

La información del participante queda actualizada y almacenada correctamente. (REF-01)

El sistema registra la acción en la bitácora de auditoría. (REF-03)

Flujo Principal (Camino Feliz)

El actor accede al módulo de gestión de participantes. (REF-01)

El actor ingresa el número de documento para buscar al participante. (REF-01)

El sistema muestra el registro encontrado. (REF-01)

El actor selecciona la opción “Editar información”. (REF-01)

El actor modifica los campos requeridos. (REF-01)

El sistema valida el formato y consistencia de los datos ingresados. (REF-05)

El sistema guarda los cambios realizados. (REF-01)

El sistema registra la actualización en el log de auditoría. (REF-03)

El sistema confirma que la información ha sido actualizada exitosamente. (REF-01)

Flujos Alternativos
A1 – Participante no encontrado

2a. El sistema no encuentra coincidencias.

2b. El sistema notifica al actor.

2c. El caso de uso finaliza.
(Basado en buenas prácticas UML – REF-05)

A2 – No se modifica ningún dato

5a. El actor no cambia ningún campo.

5b. El sistema permite cancelar sin guardar cambios.
(REF-05)

Flujos de Excepción
E1 – Error de validación en los datos

6a. El sistema detecta campos con errores (formato, valores inválidos).

6b. Muestra mensaje indicando campos incorrectos.

6c. El actor corrige y retorna al paso 5.
(REF-05 + RNF de seguridad y usabilidad — REF-04)

E2 – Fallo al guardar datos

7a. Ocurre un error interno en el sistema o en la base de datos.

7b. El sistema informa el error y sugiere reintentar.

7c. El caso de uso termina sin cambios.
(REF-05)

Reglas de Negocio

RN-05: Solo usuarios autorizados pueden modificar información de participantes. (REF-03)

RN-09: Toda modificación debe quedar registrada en el sistema de auditoría. (REF-03)

RN-10: No se permiten datos duplicados (número de documento). (REF-03)

Requisitos No Funcionales

RNF-001.1: Interfaz intuitiva que facilite edición sin capacitación extensa. (REF-04)

RNF-001.3: Operable con teclado y ratón. (REF-04)

RNF-002.3: Búsqueda extremadamente rápida por documento. (REF-04)

RNF-003.3: Protección de datos sensibles. (REF-04)

RNF-003.4: Registro de auditoría obligatorio. (REF-04)

Interfaces de Usuario

Pantalla: “Gestión de Participantes – Buscar y Editar”. (REF-02 + REF-01)

Formularios de edición con validaciones de formato. (REF-04)
