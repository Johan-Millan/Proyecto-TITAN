## Estándar de Respuesta de la API

Todas las respuestas del backend (FastAPI) siguen el siguiente formato JSON:

**Éxito:**
```json
{
  "success": true,
  "message": "Operación exitosa",
  "data": { "id_inscripcion": 1, "cupos_restantes": 9 },
  "error": null
}
```

**Error:**
```json
{
  "success": false,
  "message": "Error de validación",
  "data": null,
  "error": "El instructor seleccionado ya tiene un curso programado en este horario."
}
```

## Sprint 1 — Núcleo Operativo (18 SP)

**Objetivo:** Permitir la programación de cursos y la inscripción segura de participantes, validando cupos, disponibilidad de instructor y requisitos previos.

---

### HU-E01 — Programación de cursos

**Prioridad:** Alta | **Estimación:** 5 SP | **Sprint:** 1

**Historia:**
Como coordinador académico, quiero programar un curso (nombre, tipo, fechas, horario, instructor, sede y cupo) para ofertarlo a los participantes.

**Criterios de aceptación:**
- El sistema permite seleccionar el tipo de curso: `Autorizado`, `Reentrenamiento` o `Coordinador de Alturas`.
- El sistema valida que el instructor esté disponible (sin cruce de horario) y certificado para el tipo de curso a dictar.
- El cupo máximo no puede ser menor a 1.
- La fecha de fin debe ser posterior a la fecha de inicio.
- El sistema registra la sede asociada al curso.

**Tareas técnicas:**

*Backend:*
- Crear modelo `ProgramacionCurso`.
- Crear schema `CursoCreateSchema`.
- Crear controlador `check_instructor_availability()`.
- Crear ruta `POST /api/cursos/programar`.

*Frontend:*
- Diseñar formulario de programación con selector dinámico de instructores.
- Validar campos obligatorios antes del envío.
- Mostrar mensaje de error si el instructor tiene cruce de horario.

---

### HU-E02 — Inscripción de participantes a cursos

**Prioridad:** Alta | **Estimación:** 5 SP | **Sprint:** 1

**Historia:**
Como administrador, quiero inscribir a un participante en un curso programado para formalizar su matrícula.

**Criterios de aceptación:**
- El sistema busca al participante por número de documento.
- No se permite inscripción si el cupo está lleno.
- No se permite inscripción duplicada del mismo participante al mismo curso.
- Se descuenta 1 cupo automáticamente tras la inscripción exitosa.
- El estado del participante queda registrado como `Inscrito`.
- Se notifica al participante mediante el componente de mensajes (HU-T01).

**Tareas técnicas:**

*Backend:*
- Crear modelo `Inscripcion`.
- Crear schema `InscripcionCreateSchema`.
- Crear controlador `check_cupo_disponible()` y `check_inscripcion_duplicada()`.
- Crear ruta `POST /api/inscripciones/`.
- Implementar lógica de descuento de cupos.

*Frontend:*
- Diseñar formulario de búsqueda por documento y selección de curso.
- Mostrar cupos disponibles en tiempo real.

---

### HU-E03 — Validación de requisitos para inscripción

**Prioridad:** Alta | **Estimación:** 8 SP | **Sprint:** 1

**Historia:**
Como administrador, quiero que el sistema valide automáticamente los requisitos previos (exámenes médicos vigentes, cursos previos aprobados) para asegurar que el participante es elegible antes de inscribirlo.

**Criterios de aceptación:**
- Para cursos de tipo `Reentrenamiento`, el sistema verifica que el participante tenga un certificado previo vigente (no vencido).
- Si falta un examen médico de aptitud, el sistema bloquea la inscripción.
- El sistema muestra un mensaje de error indicando el requisito específico faltante.
- La validación se ejecuta en el servidor, nunca solo en el cliente.

**Tareas técnicas:**

*Backend:*
- Crear controlador `validate_requisitos_inscripcion()`.
- Integrar consulta de vigencia de certificados al flujo de inscripción.
- Crear ruta `GET /api/participantes/{id}/requisitos`.

*Frontend:*
- Mostrar lista de requisitos cumplidos y faltantes antes de confirmar inscripción.

## Sprint 2 — Formación y Certificación (19 SP)

**Objetivo:** Registrar la ejecución del curso (asistencia y calificaciones) y automatizar la emisión de certificados con validación de vigencia.

---

### HU-F01 — Registro de asistencia

**Prioridad:** Media | **Estimación:** 3 SP | **Sprint:** 2

**Historia:**
Como instructor, quiero registrar la asistencia diaria de los participantes de mi curso para mantener trazabilidad de la formación.

**Criterios de aceptación:**
- Solo el instructor asignado al curso puede registrar asistencia.
- El registro queda asociado a la fecha de sesión y al curso correspondiente.
- El sistema calcula el porcentaje de asistencia acumulado por participante.

**Tareas técnicas:**

*Backend:*
- Crear modelo `RegistroAsistencia`.
- Crear ruta `POST /api/cursos/{id_curso}/asistencia`.
- Implementar validación de rol (solo instructor asignado).

---

### HU-F02 — Registro de calificaciones

**Prioridad:** Media | **Estimación:** 3 SP | **Sprint:** 2

**Historia:**
Como instructor, quiero registrar las calificaciones y resultados de evaluación de cada participante al finalizar el curso para determinar si aprueba la formación.

**Criterios de aceptación:**
- El instructor puede marcar a un participante como `APTO` o `NO APTO`.
- La calificación mínima de aprobación es configurable por tipo de curso.
- La aprobación es requisito obligatorio para la generación del certificado (HU-C01).
- Un participante `NO APTO` no puede recibir certificado.

**Tareas técnicas:**

*Backend:*
- Crear modelo `Calificacion`.
- Crear ruta `POST /api/cursos/{id_curso}/calificaciones`.
- Crear controlador `check_aprobacion()` que evalúe asistencia y nota.

---

### HU-C01 — Generación automática de certificados

**Prioridad:** Alta | **Estimación:** 8 SP | **Sprint:** 2

**Historia:**
Como sistema, quiero generar automáticamente el certificado al finalizar el curso si el participante cumple con la asistencia mínima y la calificación de aprobación, para agilizar la emisión y garantizar trazabilidad.

**Criterios de aceptación:**
- El certificado se genera únicamente si el participante está marcado como `APTO` y cumple el porcentaje de asistencia mínimo requerido.
- El certificado se asocia al tipo de curso correspondiente (`Autorizado`, `Reentrenamiento`, `Coordinador`).
- Incluye un código de verificación único e irrepetible.
- La fecha de expedición y la fecha de vencimiento se calculan automáticamente según normativa (vigencia de 1 año para trabajo en alturas).
- El certificado queda en estado `Activo` al generarse.

**Tareas técnicas:**

*Backend:*
- Crear modelo `Certificado` con campos: `codigo_verificacion`, `fecha_expedicion`, `fecha_vencimiento`, `estado`.
- Crear controlador `generar_certificado()`.
- Crear ruta `POST /api/certificados/generar`.
- Implementar generación de código único (UUID o hash).

---

### HU-C02 — Validación de vigencia de certificados *(Nueva)*

**Prioridad:** Alta | **Estimación:** 5 SP | **Sprint:** 2

**Historia:**
Como sistema, quiero verificar automáticamente la vigencia de los certificados de cada participante para bloquear inscripciones con documentación vencida y alertar sobre próximos vencimientos.

> **Justificación:** En trabajo seguro en alturas, la normativa colombiana (Resolución 4272 de 2021) establece que el certificado de competencia tiene una vigencia de **1 año**. Un certificado vencido invalida la habilitación del trabajador para actividades en alturas. Este control es crítico para el cumplimiento legal del centro.

**Criterios de aceptación:**
- El sistema marca automáticamente como `Vencido` todo certificado cuya `fecha_vencimiento` sea inferior a la fecha actual.
- Un certificado `Vencido` bloquea la inscripción a cursos de tipo `Reentrenamiento` (se requiere curso `Autorizado` nuevamente).
- El sistema genera una alerta cuando un certificado está a **30 días o menos** de vencer.
- Las alertas son visibles para el administrador y para el participante.
- El código de verificación sigue siendo válido para consulta histórica incluso en certificados vencidos.

**Tareas técnicas:**

*Backend:*
- Crear tarea programada (cron job) `actualizar_estado_certificados()` que se ejecute diariamente.
- Crear controlador `check_vigencia_certificado(id_participante, tipo_curso)`.
- Crear ruta `GET /api/certificados/{id}/vigencia`.
- Integrar la validación de vigencia al flujo de HU-E03.

*Frontend:*
- Mostrar badge de estado (`Activo` / `Por vencer` / `Vencido`) en el perfil del participante.
- Mostrar alerta visual cuando el certificado esté próximo a vencer.

## Sprint 3 — Consulta y Descarga (8 SP)

**Objetivo:** Permitir a los participantes consultar y descargar sus certificados de manera autónoma.

---

### HU-D01 — Consulta de certificados por participante

**Prioridad:** Media | **Estimación:** 3 SP | **Sprint:** 3

**Historia:**
Como participante, quiero consultar el historial de certificados obtenidos asociados a los cursos que he tomado para verificar mi formación y su vigencia.

**Criterios de aceptación:**
- Se listan todos los certificados del participante con: curso, fecha de expedición, fecha de vencimiento y estado (`Activo` / `Por vencer` / `Vencido`).
- El participante puede filtrar por estado y por año.

**Tareas técnicas:**

*Backend:*
- Crear ruta `GET /api/participantes/{id}/certificados`.

*Frontend:*
- Diseñar vista de historial con filtros y badge de estado por certificado.

---

### HU-D02 — Descarga de certificado en PDF

**Prioridad:** Media | **Estimación:** 5 SP | **Sprint:** 3

**Historia:**
Como participante, quiero descargar mi certificado en formato PDF para tener un comprobante físico o digital de mi formación.

**Criterios de aceptación:**
- El PDF incluye: nombre completo del participante, tipo y nombre del curso, fecha de expedición, fecha de vencimiento, nombre del instructor, firma digital del coordinador y código de verificación.
- Solo se pueden descargar certificados en estado `Activo` o `Vencido` (para archivo histórico).
- El nombre del archivo descargado sigue el formato: `CERTIFICADO_{TIPO_CURSO}_{DOCUMENTO}_{AÑO}.pdf`.

**Tareas técnicas:**

*Backend:*
- Integrar librería de generación de PDF (ej. `reportlab` o `weasyprint`).
- Crear ruta `GET /api/certificados/{id}/descargar`.
- Generar PDF con plantilla oficial del centro.

*Frontend:*
- Botón de descarga disponible en el historial de certificados (HU-D01).
