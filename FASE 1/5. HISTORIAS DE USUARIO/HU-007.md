HU-G – Gestión e Inspección de Equipos de Seguridad

Requisitos cubiertos: RF-003.5, RF-005.3, RF-005.4

==========================================

🔹 Metadatos
ID: HU-G

Epic: Seguridad Operacional

Prioridad: Alta

Versión: 1.0

🔹 Historia de usuario
Como personal administrativo o inspector, quiero gestionar e inspeccionar equipos de seguridad, para garantizar que los elementos utilizados en las prácticas cumplan las condiciones de seguridad requeridas.

🔹 Descripción detallada
El sistema debe permitir:

Registrar equipos.

Actualizar estado operativo.

Registrar inspecciones.

Adjuntar evidencias.

Generar certificados de aptitud.

Consultar historial completo.

🔹 Criterios de aceptación
CA-01: Cada equipo debe tener código único.

CA-02: Debe mantenerse historial de inspecciones.

CA-03: Debe permitir adjuntar evidencia.

CA-04: Solo equipos aptos pueden certificarse.

CA-05: Certificados deben descargarse en PDF.

Escenarios (Gherkin)

✔ Escenario 1 – Camino Feliz: Inspección aprobada
Dado que un equipo está registrado
Cuando el inspector aprueba la inspección
Entonces el sistema actualiza el estado a “Apto”
Y genera el certificado correspondiente.

✔ Escenario 2 – Flujo alternativo: Equipo no apto
Dado que el inspector detecta fallas
Cuando registra la inspección
Entonces el sistema marca el equipo como “No apto”.

✔ Escenario 3 – Manejo de errores: Evidencia inválida
Dado que el usuario intenta subir un archivo no permitido
Cuando adjunta evidencia
Entonces el sistema rechaza el archivo
Y muestra mensaje de error.

Reglas de negocio
RN-01: Todo equipo debe tener inspección vigente.

RN-02: Equipos no aptos no pueden certificarse.

RN-03: Toda inspección requiere inspector responsable.

RN-04: Los certificados deben tener vigencia definida.

Pruebas completadas.

Historial operativo.
