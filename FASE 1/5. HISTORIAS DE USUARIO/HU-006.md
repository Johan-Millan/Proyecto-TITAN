HU-F – Control de Asistencia y Calificaciones

Requisitos cubiertos: RF-003.3, RF-003.4

==========================================

🔹 Metadatos
ID: HU-F

Epic: Ejecución Académica

Prioridad: Alta

Versión: 1.0

🔹 Historia de usuario
Como instructor, quiero registrar asistencia y calificaciones de los participantes, para llevar control del desempeño y aprobación de los cursos.

🔹 Descripción detallada
El sistema permitirá:

Registrar asistencia.

Marcar ausencias.

Registrar calificaciones.

Definir aprobado/reprobado.

Asociar resultados al historial del participante.

🔹 Criterios de aceptación
CA-01: Solo participantes inscritos pueden aparecer en lista.

CA-02: Solo asistentes pueden calificarse.

CA-03: Debe existir estado aprobado/reprobado.

CA-04: Las notas deben quedar almacenadas permanentemente.

Escenarios (Gherkin)

✔ Escenario 1 – Camino Feliz: Registro completo
Dado que el curso está en ejecución
Cuando el instructor registra asistencia y notas
Entonces el sistema almacena correctamente la información.

✔ Escenario 2 – Flujo alternativo: Participante ausente
Dado que un participante no asistió
Cuando el instructor registra asistencia
Entonces el sistema marca al participante como ausente
Y bloquea la calificación.

✔ Escenario 3 – Manejo de errores: Nota inválida
Dado que el instructor registra una nota fuera del rango permitido
Cuando intenta guardar
Entonces el sistema muestra “Calificación inválida”.

Reglas de negocio
RN-01: Solo asistentes pueden aprobar cursos.

RN-02: Las notas no pueden modificarse después del cierre sin permisos especiales.

RN-03: Toda modificación debe registrarse en auditoría.

Definición de Terminado (DoD)

Asistencia funcional.

Registro de notas operativo.

Persistencia validada.

Pruebas completadas.

Validación UX aprobada.

Notas Técnicas

Control de permisos por rol.

Validaciones automáticas.

Lista de participantes.

Campo de nota.

Estado aprobado/reprobado.
