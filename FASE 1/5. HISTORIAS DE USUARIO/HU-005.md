HU-E – Programación y Gestión de Cursos

Requisitos cubiertos: RF-003.1, RF-003.2, RF-005.1, RF-005.2

==========================================

🔹 Metadatos
ID: HU-E

Epic: Gestión Académica

Prioridad: Alta

Versión: 1.0

🔹 Historia de usuario
Como personal administrativo, quiero programar cursos y gestionar inscripciones de participantes, para organizar correctamente las capacitaciones del centro de entrenamiento.

🔹 Descripción detallada
El sistema debe permitir:

Programar cursos en calendario.

Asignar instructor, fecha, horario y cupos.

Clasificar cursos por tipo.

Inscribir participantes.

Validar requisitos según tipo de curso.

Controlar disponibilidad de cupos.

🔹 Criterios de aceptación
CA-01: Permitir programación de cursos en calendario.

CA-02: Validar disponibilidad de instructores.

CA-03: Impedir inscripción si no hay cupos.

CA-04: Validar requisitos según tipo de curso.

CA-05: Mostrar lista actualizada de inscritos.

Escenarios (Gherkin)

✔ Escenario 1 – Camino Feliz: Inscripción exitosa
Dado que existe un curso programado con cupos disponibles
Cuando el administrativo inscribe un participante válido
Entonces el sistema registra la inscripción correctamente
Y descuenta un cupo disponible.

✔ Escenario 2 – Flujo alternativo: Participante sin requisitos
Dado que el curso requiere certificado previo
Cuando el participante no cumple el requisito
Entonces el sistema bloquea la inscripción
Y muestra el mensaje “El participante no cumple los requisitos del curso”.

✔ Escenario 3 – Manejo de errores: Curso sin cupos
Dado que el curso alcanzó el máximo de participantes
Cuando el administrativo intenta agregar otro participante
Entonces el sistema rechaza la inscripción
Y muestra “No hay cupos disponibles”.

Reglas de negocio
RN-01: Ningún curso puede superar su capacidad máxima.

RN-02: Un instructor no puede tener cursos simultáneos.

RN-03: Todo curso debe tener un tipo asignado.

RN-04: Reentrenamiento exige certificado previo vigente.
